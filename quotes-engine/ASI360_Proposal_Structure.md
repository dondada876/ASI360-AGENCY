## Full Proposal — Section-by-Section Structure

### 1. COVER LETTER  _(CoverLetter.doc)_
**Purpose:** Personal introduction, sets tone, references the RFP/conversation
**ASI 360 elements:**
- Date, client name/title/company/address
- Brief statement of why we're submitting (problem summary)
- Our unique value proposition (one sentence)
- Reference to the quote number and validity period
- Primary contact: Don Bucknor, COO — 510-495-0905 — don@asi360.co
- Signature block with ASI 360 / Allied Systems Integrations logo

---

### 2. TITLE PAGE  _(TitlePage.doc)_
**Purpose:** Professional first impression, identifies the project
**ASI 360 elements:**
- ASI 360 logo + Allied Systems Integrations
- Client name + site address
- Proposal/Quote reference number
- Date of issue + valid through date
- "Prepared by: Don Bucknor, COO"

---

### 3. EXECUTIVE SUMMARY  _(ExecSummary.doc)_
**Purpose:** One-page overview — problem, opportunity, solution
**Structure (from kit):**
- **The Objective** — restate client's needs
  - Need #1, #2, #3
- **The Opportunity** — why now, what's at stake
  - Goal #1, #2, #3
- **The Solution** — what ASI 360 proposes
  - Recommendation #1, #2, #3

**Mad Oak example:**
- Need: 4+ hour footage retrieval on legacy system creates liability risk
- Opportunity: NVR swap reuses existing infrastructure at minimal disruption
- Solution: Luminys R54-32NA + LumiSearch AI phases in over 18–24 months

---

### 4. ABSTRACT  _(Abstract.doc)_
**Purpose:** One-paragraph problem/solution statement for complex proposals
**Structure:**
- Problem statement (client's specific pain)
- Market/context (why this matters for their business)
- Proposed solution (what we're doing to fix it)

---

### 5. NEEDS ASSESSMENT  _(NeedsAssessment.doc)_
**Purpose:** Demonstrate we listened — validate client's pain in their language
**Structure:**
- Needs (numbered) — the specific problems identified
- Market/constituency affected
- Solution brief
- Sources (site visit, interview, photos, assessment form)

**ASI 360 process:** Needs are gathered during:
1. Initial discovery call
2. Site assessment ($275 flat — LABOR-ASSESS)
3. Airtable intake form (photo upload)
4. ASI 360 Hub network discovery scan (Phase 1)

---

### 6. SCOPE OF WORK  _(Requirements.doc + SecurityPlan.doc)_
**Purpose:** Define what's in and what's out — protect both parties
**Structure:**
- Numbered scope items (what we're doing)
- Explicit exclusions (what's NOT in scope)
- Requirements matrix:
  - Deliverable requirements
  - Contractual obligations
  - Documentation & training requirements
  - System/hardware requirements
  - Functional requirements
  - Performance requirements

**Always include for security projects:**
- "Facial recognition is NOT included unless explicitly quoted"
- "Camera refurbishment is scoped separately after diagnosis"
- "Existing infrastructure reused unless otherwise stated"

---

### 7. TECHNICAL APPROACH  _(SecurityPlan.doc + Architecture.doc)_
**Purpose:** Show technical credibility — explain HOW we solve it
**Structure for NVR/Camera proposals:**
- Current state assessment (what they have)
- Proposed architecture (NVR model, channel count, storage, protocol)
- Hybrid camera integration approach (ONVIF/RTSP for legacy)
- Network architecture (IP scheme, static IPs, MAC binding)
- Remote access design (web + mobile, VPN if required)
- AI/analytics roadmap (LumiSearch phased rollout)
- ASI 360 Hub monitoring layer

---

### 8. PROJECT SCHEDULE — 5-PHASE FRAMEWORK  _(TimeLine.doc + InstallSchedule.doc + Milestones.doc)_
**Purpose:** Show structured delivery, set client expectations, tie to VTiger project

The standard ASI 360 5-phase framework (from Project Timeline template):

```
Phase 1: Conception & Initiation
  1.1 Project Charter — contract signed, scope confirmed
  1.2 Plan Review — site/network audit, hub deployment
  1.3 Initiation — discovery scan, IP inventory documented

Phase 2: Definition & Planning
  2.1 Scope & Goal Setting — finalize camera count, IP scheme
  2.2 Budget Approval — deposit received, equipment ordered
  2.3 Work Breakdown Schedule — install date confirmed
  2.4 Project Schedule/Gantt — delivery tracking, ETA
  2.5 Communication Plan — client update cadence
  2.6 Risk Management — contingency date, compatibility check

Phase 3: Launch & Execution
  3.1 Status & Tracking — equipment received, staged
  3.2 KPI Monitoring — pre-config complete, IP table staged
  3.3 Quality Assurance — firmware updated, units tested
  3.4 Progress Forecasts — final day-of schedule confirmed

Phase 4: Performance & Control
  4.1 Objective Execution — install day: swap, register, connect
  4.2 Quality Deliverables — all cameras live, remote access verified
  4.3 Effort & Cost Tracking — new cameras installed, AI activated
  4.4 Performance Review — alerts configured, training delivered

Phase 5: Project Close
  5.1 Postmortem — punch list, any diagnosis items
  5.2 Project Punchlist — IP map + credentials document delivered
  5.3 Final Report — sign-off, warranty activated, balance invoiced
```

**Key dates to always include:**
- Equipment order date + lead time source (ADI/Keri TX = 5 business days)
- Equipment arrival estimate
- Installation day (primary + backup)
- Project close / warranty start

### 9. BUDGET & COST SUMMARY  _(Budget.doc + CostSummary.doc)_
**Purpose:** Clear pricing with options — protect margins, justify value
**Structure:**

```
COST SUMMARY
─────────────────────────────────────
HARDWARE
  [SKU] — Description           $X,XXX
  ...
HARDWARE SUBTOTAL               $X,XXX

LABOR / SERVICES
  [SKU] — Description           $  XXX
  ...
LABOR SUBTOTAL                  $X,XXX

SUBTOTAL (PRE-DISCOUNT)         $X,XXX
ASI 360 Client Discount         -$ XXX
─────────────────────────────────────
NET TOTAL                       $X,XXX
─────────────────────────────────────
OPTION B UPGRADE (if applicable)
  Additional item                $  XXX
  OPTION B TOTAL                $X,XXX
```

**Pricing rules:**
- All hardware priced at MAP (Minimum Advertised Price) or above
- Labor priced as flat-fee per scope item (not hourly unless specified)
- Discount shown as explicit line item (not hidden in unit prices)
- 50% deposit required to order equipment
- Balance due upon completion

### 10. RISK ANALYSIS  _(Risk.doc)_
**Purpose:** Show foresight — protect ASI 360 from scope creep
**Standard risks for security installs:**

| Risk | Analysis | Resolution | Contingency |
|------|----------|------------|-------------|
| Equipment delivery delay | Distributor ships from TX — 5 business days standard | Order confirmed same day contract signed | Backup install date ±2 days |
| Legacy camera ONVIF incompatibility | Cameras >10 years may not support RTSP | Pre-verify protocol on site assessment | Manual MJPEG stream or camera replacement quoted separately |
| Network infrastructure gaps | Client switch/router may not support PoE+ | Audit during Phase 1 Hub scan | Additional switch quoted separately if needed |
| Scope expansion (client adds cameras mid-project) | Each additional camera = additional labor | Change order process documented in contract | $150/camera add-on rate |
| Camera refurbishment unknown cost | Broken camera diagnosed on-site | Flat $75 diagnosis fee, repair quoted separately | Replacement camera quoted if repair not viable |

---

### 11. DELIVERABLES  _(Deliverables.doc)_
**Purpose:** Explicit list of what client receives — used for sign-off
**Standard ASI 360 deliverables (security/NVR):**
1. New NVR installed and operational — all cameras live
2. Static IP assignment documentation (IP map: camera → IP → channel → location)
3. Login credentials document (all usernames, passwords, access levels)
4. Remote access configured — web browser + mobile app verified
5. Motion alert notifications configured to client phone/email
6. 30-minute staff training session completed
7. New camera(s) installed and integrated
8. 90-day labor warranty certificate
9. Manufacturer hardware warranty registration
10. Work order signed and closed in VTiger

---

### 12. TRAINING PLAN  _(Training.doc)_
**Purpose:** Set expectations for knowledge transfer
**ASI 360 standard training package (included in LABOR-REMOTE):**
- On-site training: 30-minute session with key staff on install day
- Topics: footage review and export, mobile app access, alert management
- Printed reference: IP map + quick-start guide left on site
- Follow-up: 30-day check-in call (included)
- Extended training available: $175/hr (LABOR-INSTALL rate)

---

### 13. WARRANTY & MAINTENANCE  _(Warranty.doc + MaintenancePlan.doc)_
**Purpose:** Define support boundaries, introduce recurring revenue
**ASI 360 standard warranty:**
- **90-day labor warranty** on all configured settings and IP assignments
- **Manufacturer hardware warranty** (Luminys: 2-year cameras, 3-year NVR)
- **ASI 360 Hub monitoring** — 6 months free, then $25/month
  - Camera health monitoring, uptime alerts, remote diagnostics
  - Power cycle management, Grafana dashboard view
  - Remote technician access via VPN

---

### 14. TERMS & CONDITIONS  _(AgreementsAndContracts.doc)_
**Purpose:** Protect ASI 360 legally — essential for BoldSign signature
**Standard ASI 360 terms:**
- 50% deposit required to schedule and procure hardware
- Balance due upon completion and client sign-off
- Change orders require written approval before execution
- Client responsible for providing site access and power
- ASI 360 not responsible for pre-existing infrastructure failures
- Proposal valid 30 days from date of issue
- All pricing subject to distributor availability

---

### 15. AUTHORIZATION / SIGNATURE PAGE
**Purpose:** The close — what BoldSign sends
**Fields:**
- Selected option checkbox (Option A / Option B)
- Client signature + printed name + date
- ASI 360 signature (Don Bucknor) + date
- Quote number reference
- Total authorized amount

## BoldSign Workflow

1. Complete proposal Word doc (full or simplified)
2. Convert to PDF or upload DOCX to BoldSign
3. Assign signer fields: client signature, client date, option checkbox
4. Add ASI 360 countersignature field (Don Bucknor)
5. Send from `documents@asi360.co`
6. On completion: update VTiger opportunity stage to "Closed Won"
7. Issue 50% deposit invoice
8. Trigger VTiger project Phase 2 (equipment order)
